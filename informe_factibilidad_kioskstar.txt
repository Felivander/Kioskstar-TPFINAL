1. DESCRIPCIÓN DEL PROBLEMA
================================================================

Los kioscos en Argentina son uno de los comercios más numerosos y de
menor digitalización del país. La gran mayoría opera sin ningún sistema
de gestión: el stock se controla a mano, las ventas no se registran
digitalmente y el dueño no tiene visibilidad real de su negocio en
tiempo real.

Esto genera tres problemas concretos:

  a) PARA EL ENCARGADO O DUEÑO:
     - No sabe cuánto stock real tiene en cada momento.
     - La carga de productos es lenta y tediosa.
     - Si tiene más de una sucursal, la gestión se vuelve caótica.
     - No tiene reportes de ventas ni rentabilidad.

  b) PARA EL EMPLEADO:
     - Herramientas poco intuitivas o inexistentes.
     - Pérdida de tiempo cargando productos manualmente.
     - No tiene una forma rápida de registrar una venta.

  c) PARA EL CLIENTE:
     - No sabe si el kiosko más cercano tiene el producto que busca.
     - Pierde tiempo yendo a un kiosko que no tiene lo que necesita.
     - No existe transparencia de stock en el canal kiosco.

================================================================
2. SOLUCIÓN PROPUESTA: KioskStar
================================================================

KioskStar es una plataforma web y mobile-first de gestión integral
para kioscos, compuesta por dos módulos principales:

  [A] PANEL DE GESTIÓN (para dueños y empleados)
      - Registro y administración de kioscos y sucursales.
      - Gestión de stock por sucursal en tiempo real.
      - Carga de productos acelerada mediante IA: el empleado saca
        una foto del producto y la IA reconoce el nombre, categoría
        y código de barras, prellenando el formulario automáticamente.
      - Registro de ventas con integración a MercadoPago para cobros
        online o en persona (QR).
      - Sistema de roles: Admin (dueño), Empleado, Cliente.
      - Dashboard con métricas de ventas y movimientos de stock.

  [B] MAPA PÚBLICO PARA CLIENTES
      - Vista de mapa interactivo con todos los kioscos de la red
        KioskStar cercanos al usuario (Google Maps API).
      - El cliente puede buscar un producto y ver qué kioscos
        cercanos lo tienen disponible en stock real.
      - Perfil público de cada kiosko con dirección, horario y
        productos disponibles.

================================================================
3. OBJETIVO DEL TRABAJO
================================================================

El objetivo del TIF es diseñar, desarrollar y desplegar KioskStar
como una aplicación Full-Stack funcional que resuelva un problema
real del sector comercio minorista en Argentina, cumpliendo los
siguientes objetivos específicos:

  - Construir un backend robusto en Node.js + Express + TypeScript
    con arquitectura en capas (controllers, routes, middlewares, services).

  - Modelar y gestionar una base de datos relacional con Prisma ORM,
    representando la lógica de negocio multi-sucursal.

  - Implementar autenticación segura con JWT, encriptación con Bcrypt
    y validación estricta con Zod, con sistema de roles diferenciados.

  - Desarrollar un frontend en React + TypeScript que consuma la API
    propia, con estado global manejado por Redux y diseño 100%
    responsive y accesible.

  - Integrar APIs externas: Google Maps para geolocalización, una API
    de visión por IA para reconocimiento de productos, y MercadoPago
    para procesamiento de pagos.

  - Desplegar la aplicación en producción: backend en Render y
    frontend en Vercel.

5. MODELO DE DATOS (TABLAS PRINCIPALES)
================================================================

  1. Users         → id, name, email, password, role, createdAt
  2. Kiosks        → id, name, ownerId (→ Users), address, lat, lng
  3. Branches      → id, kioskId (→ Kiosks), name, address, lat, lng
  4. Categories    → id, name, description
  5. Products      → id, name, barcode, categoryId, imageUrl, description
  6. Stock         → id, branchId (→ Branches), productId (→ Products), quantity
  7. Sales         → id, branchId, userId, total, paymentMethod, createdAt
  8. SaleItems     → id, saleId (→ Sales), productId, quantity, unitPrice

Total: 8 tablas con relaciones bien definidas.
Supera ampliamente el mínimo de 4 tablas requerido.

8. PROPUESTA DE VALOR Y VISIÓN DE PRODUCTO
================================================================

KioskStar no es solo un TIF. Es la base de una red de kioscos
digitalizados, donde cada comercio que adopta el sistema aporta
transparencia al ecosistema y los clientes se benefician de poder
encontrar lo que buscan sin perder tiempo.

La visión a futuro incluye:
  - App nativa mobile (React Native).
  - Sistema de fidelización para clientes frecuentes.
  - Integración con proveedores mayoristas para reposición automática.
  - Analítica de demanda por zona geográfica.

Para el TIF, el foco está en la solución funcional que valide la
hipótesis central: un kiosko digitalizado es más eficiente y un
cliente informado es más fiel.
